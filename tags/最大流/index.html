HDU3081 Marriage Match II

##题意
有n个女孩,n个男孩
女孩对男孩存在认同关系,某女孩认同某男孩
女孩之间存在可传递的朋友关系,某女孩和某女孩是朋友
然后他们要开始玩游戏了
每轮游戏,每个女孩都要选择一个男孩当py,并且有如下要求
1.这个男孩必须是女孩认同的,或者是这个女孩的朋友认同的
2.这个男孩不能是女孩之前选过的
问最多可以进行多少轮游戏

##解题思路
首先对于这种可传递的朋友关系
第一反应是用并查集来维护这些关系
然后建图
源点连向每个女孩,容量为X
每个女孩连向可选的男孩,容量为1
男孩连向汇点,容量为INF
当且仅当最大流为满流时(最大流等于X*n),表示可以进行X轮游戏
于是二分X即可

POJ2711 Leapin' Lizards

##题意
若干只蜥蜴在柱子上,柱子下面是火海
蜥蜴的跳跃范围有限
每个柱子都有高度,并且蜥蜴从柱子上起跳后,柱子的高度会下降1单位
柱子的高度为0后变不可用
并且每个柱子上只能存在一只蜥蜴
蜥蜴需要通过跳跃在柱子之间移动,并最终跳出地图
求最少有多少只蜥蜴不能逃离

##解题思路
把柱子拆点,并连上容量为高度的边
把距离在跳跃范围内的柱子都连上边,容量INF
若可以从某个柱子跳跃到地图外,则该柱子连到汇点上,容量INF
若某个柱子上又蜥蜴,则汇点连到该柱子上,容量1
求最大流即可
需要注意的是,输出的是不能逃离的数量,所以要用蜥蜴数量减去最大流
并且输出的时候要注意were和was,lizard和lizards

HDU3416 Marriage Match IV

##题意
有向图给出两点A和B,并且每条边只能走一起
求从A到B得最短路的个数

##解题思路
由于每条边只能走一个,所以用网络流来解此题
首先找出所有位于最短路上的边
由于所有边都在最短路,所以随便找一条从A到B的路,便是最短路
于是问变为如何找出最多的从A到B的路,使每条边只经过1次
于是用这些边重新建图,每条边的容量为1
源点为A,汇点为B,最大流即为最短路的个数
省下的问题是如何找位于最短路上的边
在原图上以A为源点求最短路,得到dist1
在反图上以B为源点求最短路,得到dist2
然后遍历每条边,若dist1[u]+dist2[v]+w==dist1[B]成立
则这条边就是位于最短路上的边,丢到网络流的图中即可

HDU4289 Control

##题意
N个城市由M条高速公路连接
恐怖分子要从城市A把大规模杀伤性武器运输到城市B
为了监视恐怖分子,你需要选择一些城市安插人员来进行监视
每个城市都可以监视到所有连接这条城市的高速公路
在每个城市安插人员,都需要不同的花费
选择一些城市,在保证能够监视到恐怖分子的前提下,使费用最小

##解题思路
首先把每个城市拆为两个点,两点之间连边,权值为花费
这样就变成了从拆点的边中选择总权值最小的边集,使得从A通往B,必将经过这些边
也就是最小割,跑最大流即可
建图时记得把其他边的容量设为INF

HDU3605 Escape

##题意
2012地球要完了,所以地球上的n个人要逃离地球
总共有m个星球,但是某些人不适合居住在某些星球上
并且每个星球能够容纳的人数有限
给出每个人能够居住的星球
求能否所有人都逃离地球

##解题思路
看似是很裸的最大流
源点连到每个人上,容量1
每个人连到能居住的星球上,容量1
每个星球连向汇点,容量为容纳上限
若最大流等于人数,则YES,否则NO
但是这么做会华华丽丽的TLE
观察一下数据范围
(1 <= n <= 100000), (1 <= m <= 10)
星球数量m很小,于是不同的可居住组合也不过2^10=1024,可以很方便的用二进制表示
于是把具有相同选择的人并到一起
若有y个人的可居住组合都是x,则源点连向x,容量为y,x连向可居住的各星球,容量y
每个星球还是连向汇点,容量为容纳上限

POJ3436 ACM Computer Factory

##题意
一台计算机由p个部分组成
有n个处理半成品的机器
每个机器都各自的生产效率,每小时能够处理多少台
并且都对半成品有要求:
必须已经完成某部分,必须没有完成某部分,无所谓是否完成某部分
满足要求的半成品被加工后,会被处理成统一的半成品
求每小时最多能生产多少台成品
即从p个部分都没完成到p个部分都完成
并输出方案,A机器每小时把W台半成品送到B机器上继续加工

##解题思路
最大流
先把每台机器拆为两个点,容量为每小时处理的数量,保证流过该机器的流量不会超
若某机器可以从0生产(即可以接收所有部分都没完成的产品),则源点连向该机器,容量INF
若某台机器可以生产处成品,该机器连向汇点,容量INF
若i机器可以接收j机器处理完的半成品,则j连向i,容量INF
完事跑最大流,输出流量不为0的边即可
由于p<=10,所以半成品的状态可以用二进制来表示
这样判断某机器是否接收某半成品,可以直接用二进制运算来判断

POJ1087 A Plug for UNIX

##题意
共有N的插座,第i个插座的型号为Ri
M个设备,第i中设备需要使用Di种型号的插座
K种转接头,第i中转接头可以把需要使用A1i种插座转换为A2i种,每种都有无数个
求最少的不能使用插座的设备

##解题思路
本题需要做一种合理的安排
即把某些设备知己插在插座上,某些设备使用若干个转接头后再插到插座上
我们把着三种东西都看成点,以[可以插]的关系,将他们连起来
源点连接每个设备,容量为1
每个设备首先连向可以插入的插座,容量1
每个设备再连向可以插得转接头,容量1
每个转接头连向可以插的插座,容量1
每个转接头连向可以插得转接头,容量INF
每个插座连向汇点,容量1
这样每1单位流量从源点流入汇点,就表明有一种设备插到了插座上
转接头->转接头的容量为INF,因为转接头可以无限用
其余边容量为1,保证了其余设备仅被1单位的流量流过,也就是使用一次
最后用设备总数减去最大流即可

POJ3281 Dining

##题意
FJ给N只奶牛准备了F种食物和D中饮品
但是每只奶牛都只吃某些食物和饮品
奶牛需要食物和饮品,只有同时得到这两样,才能被满足
求最多的被满足的奶牛的数量

##解题思路
由于奶牛需要食物和物品
于是我们让同时满足这两种物品时,流量为1
把每个食物连向吃它的奶牛,奶牛再连向喝的饮品
源点连向所有食物,所有饮品连向汇点
容量都为1
这样从源点发出流量,经过食物,经过奶牛,经过饮品后到达汇点
由于容量都是1,所有到达汇点的容量表示被满足奶牛的个数
不过这样建图还有些问题
当多个食物流向同一只奶牛,然后再流到多个饮品上
这种情况是,一只奶牛被满足的多次
而我们需要的是,尽可能满足更多的奶牛,所以这样得到的答案显然不对
解决方法是把奶牛进行拆点,两个点直接连一条容量为1的边
这样就能保证每只奶牛最多被满足一次

ZOJ3229 Shoot the Bullet

##题意
需要在n天内给m个妹子拍照
第i个妹子至少要拍够Gi张
对于第i天,需要给Ci个妹子拍照,并且拍照总数不能大于Di
给某妹子拍照的数量必须在一定的范围内
求最多能拍多少张照片

##解题思路

###建图
把天和妹子看成点
源点连接每一天,容量下界为0,上界为Di
每个妹子连接汇点,容量下界为Gi,上界无限
然后把每天和能拍照的妹子连起来,容量为给妹子拍照的范围
建好图,求最大流即可
一开始是我建成了源点->妹子->天->汇点
但这样WA,反过来就AC,我也想不通为什么

###有源汇有上下界最大流的求法
首先转化为无源汇有上下界网络流:
然后添加一条从汇点到源点,容量为无穷的边
把原图看做无源汇的网络,即源点和汇点看做普通点
接着按照无源汇的方法,建立超级源点SS,超级汇点TT(为了和原图的源点和汇点区分开)
degree[i]为下界容量出之差
SS连接至degree[i]>0的点,TT连接至degree[i]<0的点
若SS到TT得最大流等于所有degree[i]>0之和,说明最大流存在
删掉超级源点和超级汇点(以及与其相连的边)
从原图源点s到汇点t的最大流便是答案
