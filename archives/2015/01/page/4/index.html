POJ3683 Priest John's Busiest Day

一天中有n个婚礼要举行
第i个婚礼的时间是Si-Ti
每个婚礼都需要神父的祝福
第i个婚礼需要Di时间的祝福
神父只能在婚礼开始时或婚礼结束前祝福
也就是说,神父只能在Si~Si+Di或Ti-Di~Ti这两个时间段出现
并且需要注意,神父不能同时参加两场婚礼
如果神父可以做到,输出YES,并输出时间安排
如果做不到,输出NO

对于每场婚礼,只能在开头参加或结尾参加,于是很明显用2-SAT可解
于是第一步建图,找出冲突关系,也就是判断时间的相交情况
把时间看作线段,比如下面两场婚礼x表示参加时间

若在i婚礼结束前参加,则必须在j开始时参加
若在j婚礼结束前参加,则必须在i开始时参加
于是这就是两条边
类似的,总共有4中相交的情况
另外判断is~it,js~jt两条线段相交的写法是
if(jsis)
因为这个,我为POJ贡献了不少WA
建好图,Tarjan走起求强连通分量
这题还需要输出一组解,大概思路如下
如果选择点Si,则与Si相连的点都要选择,并且┐Si及┐Si相连的点都不选
首先把强连通分量缩缩点,因为强连通分量具选了其中任意一个就都要选的性质
若选择Si,则要选择Si的所有后代,这样略麻烦
所以每次都选择无后代的节点,也就是出度为0的节点Si,并不选择┐Si

这样很容易想到把G'的边全部反向得到G''
然后对G''进行拓扑排序并,并按照拓扑序进行染色
由于2-SAT的对称性,同时染色即可

POJ3207 Ikki's Story IV - Panda's Trick

题目描述简直丧心病狂,总之大概题意是这样的
圈上有n个点,有m对连线连接着其中的一些点
线只能在圆外或圆内
问有没有方法让这些线不相交
首先要把目标转向连线,而不是点
什么情况下两条线会相交呢?

这种情况时,如果两条线在同测,则一定会相交
换句话说,如果两条线段是这样的关系,那么一定要给把他们分开
即一条在外,一条在内
写到这里,似乎用并查集也可以解决的,不过先继续说2-SAT的解法

#2-SAT解法
出现这种情况就要添加四条边
内i->外j
外j->内i
内j->外i
外i->内j
也就是分别以i和j进行考虑,建双向的边
建好图后,2-SAT走起就OK了

#并查集解法
用并查集表示非集
把边扫一遍
把不能在同测的边进行拆分
拆分时如果不能拆,则会相交
如果所有的边都可拆分,则不会相交
用并查集实现,代码更少,思路更清晰

POJ1515 Street Directions

即把所有无向边改为有向边
求最多能把多少无向边改为有向边

根据邻接表建无向图,一般都是建成双向的有向图
所以标记走过的边时,要把u->v和v->u都标记为走过的

void dfs(int u,int p)
{
    dfn[u]=low[u]=++dcnt;
    st.push(u);
    for(int i=ehead[u];i;i=edge[i].next)
    {
        int v=edge[i].v;
        if(v==p)continue;
        if(!dfn[v])
        {
            dfs(v,u);
            low[u]=min(low[u],low[v]);
            if(dfn[u]<low[v])
            {
                int x;
                do
                {
                    x=st.top();st.pop();
                    bcc[x]=bcccnt;
                }while(x!=v);
                bcccnt++;
            }
            
        }
        else
            low[u]=min(low[u],dfn[v]);
    }
}
int n;
void Tarjan()
{
    for(int i=1;i<=n;i++)if(!dfn[i])
    {
        dfs(i,0);
        while(st.size())
        {
            bcc[st.top()]=bcccnt;
            st.pop();
        }
        bcccnt++;
    }
}

这种方法有些麻烦,需要在两个位置对栈进行操作
其实还有更简洁的方法,判断dfn[u]==low[u]
网上的代码基本都是用的这种方法
low[u]表示u所能到达的节点的最小的dfn
如果low[u]dfn[u],则不会形成包括u的环路
如果low[u]==dfn[u],则表示通过u的子节点还能回到u
于是包括u在内的u的子节点就是一个完整的环路
也就是双连通分量

POJ3177 Redundant Paths

给出无向图,求出最少增加多少条边后会变为双连通图
双连通分量中,已经是每个点都可互达
于是可以用缩点的思想,把双连通分量进行缩点
缩点后图会变为树
剩下的问题就是最少加多少条边,树会变为双连通图
只要把入度为1的点之间连一条边,那么就会形成环路
一条边可以解决掉2个度数为1的点
于是求出度数为1的点的个数deg1
答案便是deg1/2
但是这种解法有个小BUG
如果只有1个入度为1的点
1/2==0很显然是错的
推广一下,deg1为奇数时,需要(deg1+1)/2
偶数+1,由于是整除,对结果也不会有任何影响
于是正解为(deg1+1)/2
还有个问题需要注意的是,题目中说了,可能会出现重边
于是使用邻接表加边时,需要判重
顺便,POJ3352和本题唯一的不同之处就是POJ3352保证没有重边
于是直接用本题代码提交到POJ3352是完全可以AC的

POJ2942 Knights of the Round Table

题意:有n个骑士,骑士们需要坐在圆桌上开会,开会必须满足两个条件
1.互相憎恨的骑士不能坐在一起
2.开会的骑士必须是奇数,而且左右都要有人
不能参加任何会议的其实要被开除,请求出最少要开除多少骑士

这题的知识点蛮多的

  • 求双连通分量
  • 判断奇环
  • 用交叉染色法判断二分图

不得不吐槽一下某本书,把这么复杂的题作为第一道双连通分量的练习题
数据给出的是骑士的憎恨关系
很容易想到建反图,即把可以坐在一起的骑士连上线
然后求出所有能坐在一起开会的骑士,判断是否是奇数
双连通分量的性质是,分量重任意两点都有至少2条不同的路可互达
也就是说,分量内的任意两个骑士都能一起开会
骑士坐在圆桌上,还必须是奇数,于是就是判断双连通分量内是否存在奇环
双连通分量中,如果有一个点在奇环内,则整个双连通分量的所有点都在奇环内
简单证明一下
一定能找到一条经过所有点的环
如果是奇环,则所有点都在奇环内
如果是偶环,那么其中任意一点在其他的奇环中,剩下的点也可以构成奇环

节点数为6,偶数
不过其中1,2,3形成了一个奇环
剩下的自然也构成了奇环
判断奇环可以用交叉染色法
若u染为0,则与u相邻的所有v节点都染成1,与v相邻的所有节点再染成0
只然0和1两种颜色
若整个双连通分量都可染成两种颜色,也就是整个双连通分量为二分图
那么这个图就不存在奇环
若存在奇环,则一点会出现染色失败的情况
总结一下
先建反图
然后找双连通分量
用交叉染色法对每个分量进行染色
如果染色失败,则该分量中存在奇环,则该分量的所有点都在奇环中,这个分量的所有骑士都可以参加会议
最后统计出不能参加会议的骑士数,输出即可
求双连通分量时,需要注意,判u是否为断割点的条件有两个
1.u为根节点,并且u的子树大于1
2.u不为根节点,dfn[u]<=low[v]
如果没有边从u的子树返回到u,则u不可能属于分量
如果有边从u的子树返回到u,则一定可以通过第二个条件把u加入到分量中
所以求双连通分量时,为根节点的割点要无视
在实现上就更方便了,不需要对根节点作特殊处理了
