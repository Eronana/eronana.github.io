POJ1276 Cash Machine

##题意
共有n种不同面值的钞票,第i种有Ni个,面值为Di
求这些钞票能够组成的不超过cash的最大值

##解题思路
就是多重背包,背包大小=cash,物品价值=物品体积=Di
解决多重背包问题的方法,通常是转化成01背包或完全背包

###转化为01背包
若某物品的数量是Mi,价值为Wi,体积为Ci
则需要通过二进制思想,将其分解为log(n)个单独的物品
比如n=13时,分解为
1*Wi,1*Ci
2*Wi,2*Ci
4*Wi,4*Ci
6*Wi,6*Ci
这四个,可以组成选择1-13个里的任意一种
然后跑一遍01背包即可
复杂度为O(N*V*Sum(logMi))

###转化为完全背包
直接转换成完全背包,就有种暴力的味道了
每个物品取的个数为1-Mi,跑Mi次完全背包,取最大值
于是这样复杂度就略高了
最外层n个物品,最内侧是背包体积V,中间层,每次Mi
于是复杂度为O(N*V*Sum(Mi))
这种复杂度我表示不能接受,还不如拆成01背包呢
于是我找到了O(N*V)的方法
网上说的O(N*V)方法都是用单调队列,好像很麻烦的样子
不过我发现某些代码也确实是O(N*V),但是和单调队列的方法又不一样
其思路是这样的
开一个数组,记录跑完全背包时,每个状态所用当前物品的次数
每次跑完全背包前,该数组先清零
只有当该数组记录的次数小于物品个数,并且满足状态转移条件时,才能够进行状态转移,并维护该数组
用这种方法,只需多开一个数组(大小为背包容量),即可实现O(N*V)的复杂度

CF466D Increase Sequence

##题意
给出一个序列a1,a2,...,an
你可以选择一个区间,把这个区间内的数全部+1
选择的区间不能包含相同的起点或相同的终点
比如选择[2,5]和[2,8]是不允许的
[4,9]和[2,9]这样终点相同的也不允许
求有多少种方法能让序列中所有的数都等于h

##解题思路
每个数,都可以有以下几种操作
[ 作为某个区间的开始
] 作为某个区间的结束
[]选定这个数作为长度为1的区间
][作为区间的结束和开始
- 什么也不做
另dp[i][j]为第i个数后面有j个未闭合区间的个数
状态转移方程有两种

###第一种
a[i]+j==h的情况
j个些区间,每个都会产生+1的作用,在加上a[i]就与h相等
于是第i个数有可以由两种操作的使其等于h
1.什么也不做,有a[i-1][j]种
2.在a[i-1][j-1]的基础上,在i上添加一个[,有a[i-1][j-1]种
于是方程为a[i][j]=dp[i-1][j]+dp[i-1][j-1]

###第二种
a[i]+j+1==h的情况
和上个情况相比,少了一个未闭合区间(要+1才==h)
于是需要在i上加一个[
但是加上[,i后面的就是j+1个未闭合区间了,所以还要在i再添加一个]
添加的],可以和前面j个中和i上添加的任意一个[组成区间,于是有
dp[i-1][j]*(j+1)种
其实还可以取dp[i-1][j+1]
不过直接取得话,就是a[i][j+1]了,所以我们要再i上添加],闭合一个区间
至于这个]是和哪个区间闭合的呢?总共有j+1个区间,于是有dp[i-1][j+1]*(j+1)种情况
于是有方程a[i][j]=dp[i-1][j+1]*(j+1)+dp[i-1][j]*(j+1)
实现上,注意范围,用long long
并且第一种情况的a[i-1][j-1]要加上判断,j不能为0
再就是dp的范围
第一层i是从1-n,这个没什么好说的
第二层可以由两种情况的条件推出
a[i]+j==h -> j=h-a[i]
a[i]+j+1==h -> j=h-a[i]-1
于是范围是h-a[i]-1到h-a[i]
然后还要注意边界情况,最小为0,最大为i,套个max和min即可

POJ1837 Balance

##题意
有个天平,共有C个位置可以挂砝码
位置为负表示在左边,为正表示在右边
共有G个砝码
求有多少种使天平平衡的挂法

##解题思路
题目中已经用正负来区分转轴的左右了
要使天平平衡,便是使左右力矩相等,也就是合力矩为0
于是转化为有多少种方法使合力矩为0
我们用01背包的思想来考虑这个问题
令dp[i][j]表示前i个砝码组成力矩j的个数
如果选第i个砝码,则第i个砝码挂到C各位置上,可以组成C个力矩
设第i个砝码挂到第j个位置上的力矩为Mi
很容易得出方程dp[i][j+Mi]+=dp[i-1][j]
于是推一遍,dp[G][0]便是答案
实现上,首先考虑力矩的范围
每个砝码为1-25,最多有20个砝码
挂钩的位置从-15到15
于是力矩的距离为[-15*25*20,15*25*25]
注意这里出现了负数,所以在实现上转化为处理[0,15*25*25*2]
答案为dp[G][15*25*25]
再就是需要判断dp[i-1][j]是否为零
如果为零,则直接跳过
否则会进行这样无意义的计算
dp[i][j+G[i]*C[k]]+=0;

NOIP2007T3 矩阵取数游戏

##题意
给出n行m列的矩阵a
每次从每列中取走一个数,共取m次
规定最能取最前面的或最后面的,得分为去到的数*2^i.i为次数,从1到m
求最大分数

##解题思路

###DP
首先很容易观察出,每行都是独立的
这样只关注如何使每行的分值最高即可
解决的方法便是DP
设dp[i][j]为区间[i,j]的最大得分,于是很容易得出状态转移方程
dp[i][j]=max(a[x][i]+dp[i+1][j],a[x][j]+dp[i][j-1])
我对于这个方程的理解是这样的:
dp[i][j]可以由两种子状态得到,dp[i-1][j]和dp[i][j-1]
也就是去左边缺少i和右边缺少j的情况
然后再补充上i和j的,去最大值,便是dp[i][j]
现在的方程还缺少点东西,因为分数要2^i,现在加上,方程变成
dp[i][j]=2max(a[x][i]+dp[i+1][j],a[x][j]+dp[i][j-1])
直接在前面*2即可

###大整数
由于本题数据量略大,远远超出可用类型的范围
所以需要写大整数
大整数一般比赛时写成函数会比较快
平时刷题时,用重载了各种运算符的大整数类比较方便
不过我这次作为练习,手撸了个仅支持加法和int乘法的大整数类
用类的最大的好处就是,只需改变变量类型,其他的部分完全不用动
当然,前提是必须做好相应的运算符重载

CF510D Fox And Jumping

#题意
一条有无数个节点组成的带子,第一个断点编号0,第二个1..以此类推
有n个卡片,每个卡片有两个属性li和ci
购买第i个卡片需要花费ci,使用第i个卡片,可以向前或向后跳跃li个节点
卡片购买后就可以使用任意次
求最小花费多少,可以达到任意一个节点.若不可能,输出-1

##解题思路
要想达到任意一个节点,就必须从i能到达i+1
也就是说,必须能向前跳1个节点
不难发现,使用两张卡能够跳跃的最小值为GCD(la,lb)
也就是说GCD(la,lb)=1,则使用a,b两张卡可以达到任意节点
这个结论很容易推广到k张卡片上
于是就转化为选择k张卡片,是这些卡片l的GCD=1,并且花费最少
只要求出所有组合的gcd,找出gcd=1并且花费最少的即可
用DP即可

POJ3659 Cell Phone Network

n个点,n-1条边构成的树,最小支配集
数的最小支配集一般有两种方法,贪心和DP
这里使用的是DP
状态有三种:

  • dp[u][0] i点属于支配集
  • dp[u][1] i点不属于支配集,但被子节点覆盖
  • dp[u][2] i点不属于支配集,且不被子节点覆盖

还有这三种状态都表示以u为根的子树都被覆盖
于是状态转移方程

  • dp[u][0]=sum(min(dp[v][0],dp[v][1],dp[v][2]))+1
  • dp[u][1]=sum(min(dp[v][0],dp[v][1]))
  • dp[u][2]=sum(dp[v][1])

dp[u][1]中需要注意两点

  • 如果u没有子节点,则dp[u][1]=INF
  • 如果min取到的都是dp[v][1]的话,dp[u][1]就不会被覆盖

所以一旦出现这种情况,就需要选一个dp[v][0]来覆盖[u][1]
选哪个,当然是选择相差最小的,也就是min(dp[v][0]-dp[v][1])
由于dp[v][1]可能为INF,所以dp[u][2]要判断一下,不然太多INF相加肯定会悲剧的
