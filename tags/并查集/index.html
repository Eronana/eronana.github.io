关于并查集的初始化

主流的并查集是这样初始化的

然后Find函数这样定义
int Find(int a)
{
return Father[a]==a?a:Father[a]=Find(Father[a]);
}
一开始我也这么用,直到某天突然发觉,可以这样初始化

然后Find函数这样定义
int Find(int a)
{
return Father[a]==-1?a:Father[a]=Find(Father[a]);
}
也就是说,改变了根的标识
前者i为根时,Father[i]==i
二后者i为根时,Father[i]==-1
于是通过这样的小修改,便可以让初始化简单一点
不过也带来了一些不方便的地方
如果用前者,在全部合并操作完成后,对所有元素查找一遍

然后就可以用Father[i]来表示i的根了
理由很简单,如果i是根,那么Father[i]一定是i
如果i不是树根,那么经过之前的Find(i)路径压缩后,Father[i]便是i的根
不过这种技巧对于后者来说则行不通
因为后者用Father[i]==-1来表示i是根

POJ3694 Network

##题意
给一个无向图,动态的加边,输出每次加边后桥的数量

##解题思路
先用Tarjan求出所有桥,并把双连通分量缩点,于是原图就变成了树
如果加边不能减少桥,那么这条边的两个点一定位于同一个分量中
如果可以减少桥,那么在树上肯定会形成环
换上的边都是减少的桥,只要找出这些边即可
很容易得出,添加u->v边后形成了环,那么这个环的最上方一定是lac(u,v)
用样例2来举例

在3,4直接连边,3,4,lca(3,4)=1会形成环
在考虑缩点的话,1 2 3直接缩成一点,于是把1,2,3都当成1
3,4连边,相当于1,4连边,于是1,4,lca(1,4)=1形成环
只要动态的找出需要连边的u,v两点的LCA,并相应的缩点即可
实现上,有两种方法
一种是不缩点,直接在原图上进行LCA
另一种是缩点,在求桥时,直接用并查集把在同一分量中的点并在一起,并切要让最上方的点为根
求LCA时,用并查集就可以快速找到当前分量的最上方的点(并查集的根)
然后向上回溯即可

POJ2912 Rochambeau

##题意
N个熊孩子玩剪刀石头布
其中一个是裁判,然后剩下的熊孩子分成三组(有可能某组是空的)
每个组的每个人都一直使用同一个手势
并且每个组使用的手势都不同
裁判和任何人玩,手势都是随机的
给出熊孩子们的结果
求最少用多少条结果就能找出谁是裁判
如果不可能找出,输出Impossible
如果找不出,输出Can not determine

##解题思路
根据已知条件,可以得出同组的结果一定是平局,不同组的每次输赢相同
也即是三个组相互克制
而裁判则是个奇葩,表现出飘忽不定的情况
也就是如果假设裁判也位于某组的话,则可能出现矛盾的情况:
和同组比赛出不同手势,和非同组比赛出相同手势
分析出这些,就很容易用并查集结局了
用带权并查集来维护组别的关系:
若AB同组(=),则dist(a,b)=0
若A赢了B(>),则dist(a,b)=1
若A输了B(<),则dist(a,b)=2
查询时对距离取模,即可获取两组的关系
最后枚举裁判,把裁判不参与的比赛用并查集维护
如果有多余1个没有矛盾的情况,则表示还不能判断出哪个是裁判
如果全部都矛盾,则表示条件有问题,不可能找出哪个是裁判
至于找出最少条判断出裁判的结果,只需维护最大的矛盾编号即可

POJ1456 Supermarket

##题意
有N个商品,每个商品都有利润和保质期,每天只能卖一种商品
过期的商品不能出售,问怎么卖可以使总利润最大

##解题思路
贪心,按商品利润排序,从利润最大的开始卖
每个商品,尽量在最后一天卖掉,这样可以争取时间卖更多的商品
如果最后一天已经卖了其他商品了,就把时间往前提,直到可以卖
这种贪心策略是正确的,于是剩下的问题是如何快速找出可以出手某商品的日子
我们用并查集维护天数,根节表示可出售的日子
卖掉某商品后,把根节点r和r-1并在一起
表示可出售的日期往前提一天

ZOJ3261 Connections in Galaxy War

##题意
有N个星球
给出各星球的战斗力,以及星球之间的隧道
星球之间可以直接通过隧道通信,也可以通过可以通信的星球进行中转
动态的摧毁隧道
以及动态的查询某星球能够联系上的战斗力最高的星球
如果找不到战斗高于自身的星球,则输出-1
战斗力最高的星球多余一个,输出序号最小的

##解题思路
由于可通信的关系是可以传递的
于是可以用并查集把可互相通信的星球并在一起,并让战斗力最高的星球为根
这样查询时直接输出根节点即可
不过另一个操作并不是合并,而是摧毁,即拆除关系,并不能直接用并查集搞定
解决方法是全部读入后离线操作
先把除了会拆除的隧道先并起来
然后将所有操作都倒过来
查询还是查询,而摧毁改为合并
利用这种离线倒置的方法,就可以很容易的解决这种拆分操作的并查集

POJ2492 A Bug's Life

##题意
教授Hopper在研究虫子的性行为
他发有些虫子在嘿咻,然后做了很多如下格式的记录
a虫子和b虫子在嘿咻
众所周知,正常来说只有异性之间才能嘿咻
于是你需要找出这些虫子中是否有同性恋

##解题思路
很典型的并查集题
可以用距离来描述虫子直接的关系
两只在嘿咻的虫子就并在一起,并且直接的距离为1
如果两只嘿咻的虫子的距离%2为0,就表明这两只里有一只是同性恋
另一种方法是开两个并查集来表示非集关系
a和b合并时,a和b的非集合并,b和a的非集合并
如果a和b在同一集合或a的非集和b的非集在同一集合,就表明存在冲突
实现上,两个并查集可以用一个二倍的并查集代替,实现上也方便了很多啊

HDU3038 How Many Answers Are Wrong

##题意
有一个长度为n的序列,然后给出m条格式如下的信息
第i个数到第j个数的和是s
你需要统计出与之前信息矛盾的信息的个数,并忽略矛盾的信息

##解题思路
这是一个带权并查集的典型应用
我们把序列上的每个数,看做是距离
则对于第i个数Ai就表示从i到j+1的距离
而从第i个数到第j个数的和,就是第i到j+1的距离

如上图,第1个数为7,表示1到2直接的距离
第一个数7加上第二个数3=10
表示1到2+1=3的距离
做了这样的变换以后,就很容易用并查集处理了
用并查集来表示线段
若a和b在同一集合中,则表示a和b在同一线段中,并且存在相对距离
回到题目上,什么样的信息算是矛盾的呢
从a到b的和是s
如果a和b不在同一个集合,则需要把a和b并在一起,并维护距离
若a和b已经在同一个集合,就要判断a到b+1的距离是否为s
如果不为s,则说明这是一条矛盾的信息
带权并查集建立在普通并查集上,只需做一些修改
我们加入数组dist,dist[a]表示a到父节点的距离
和并a和b时,通常我们需要找到a的祖先x,b的祖先y,并且令x的祖先为y
接下来,就要维护dist了

已知a到b的距离为c,a到x得距离dist[a],b到y的距离dist[b],求x到y的距离
很容易推导出,dist[x]=dist[b]-(dist[a]-c)
于是合并时,距离的维护就完成了
最后是并查集的精髓,查找时的距离维护
由于路径压缩,在查询a后,a的父节点已经变为了根节点
dist[a]+=dist[a的父节点的新距离]
所以在查找a的父节点前,记录一下a的父节点
查找完维护即可
