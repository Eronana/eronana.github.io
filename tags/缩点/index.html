POJ3694 Network

##题意
给一个无向图,动态的加边,输出每次加边后桥的数量

##解题思路
先用Tarjan求出所有桥,并把双连通分量缩点,于是原图就变成了树
如果加边不能减少桥,那么这条边的两个点一定位于同一个分量中
如果可以减少桥,那么在树上肯定会形成环
换上的边都是减少的桥,只要找出这些边即可
很容易得出,添加u->v边后形成了环,那么这个环的最上方一定是lac(u,v)
用样例2来举例

在3,4直接连边,3,4,lca(3,4)=1会形成环
在考虑缩点的话,1 2 3直接缩成一点,于是把1,2,3都当成1
3,4连边,相当于1,4连边,于是1,4,lca(1,4)=1形成环
只要动态的找出需要连边的u,v两点的LCA,并相应的缩点即可
实现上,有两种方法
一种是不缩点,直接在原图上进行LCA
另一种是缩点,在求桥时,直接用并查集把在同一分量中的点并在一起,并切要让最上方的点为根
求LCA时,用并查集就可以快速找到当前分量的最上方的点(并查集的根)
然后向上回溯即可

POJ3592 Instantaneous Transference

##题意
1 //一组数据
2 2 //下面是2*2的地图
11 //数字代表该点的矿物数量
1* //*代表传送点
0 0 //传送点的目标坐标
地图中可能有多个传送点
地图上还可能有#,表示不能进入的区域
从左上角开始走,只能向下走或向右走
到达的点上的的矿物会被取走
到达传送点,可以选择传送到目标坐标,或者继续向下或向右走
问最多能挖到多少矿物

##解题思路
如果没有传送点,本题就是一道十分简单的搜索(DP)
因为存在传送点,就构成了环路,所以不能直接搜索或DP
很容易想到,既然形成了环路,那么环路上的点一定都可以取到,并且到达环路上的任意一点
而且多个环路套在一起也是如此,于是很容易就能想到缩点
先求出强连通分量,然后把每个分量缩成一点,矿物数量为分量所有点之和
缩点后,环路自然消失,可以安心的搜索/DP了

HDU3605 Escape

##题意
2012地球要完了,所以地球上的n个人要逃离地球
总共有m个星球,但是某些人不适合居住在某些星球上
并且每个星球能够容纳的人数有限
给出每个人能够居住的星球
求能否所有人都逃离地球

##解题思路
看似是很裸的最大流
源点连到每个人上,容量1
每个人连到能居住的星球上,容量1
每个星球连向汇点,容量为容纳上限
若最大流等于人数,则YES,否则NO
但是这么做会华华丽丽的TLE
观察一下数据范围
(1 <= n <= 100000), (1 <= m <= 10)
星球数量m很小,于是不同的可居住组合也不过2^10=1024,可以很方便的用二进制表示
于是把具有相同选择的人并到一起
若有y个人的可居住组合都是x,则源点连向x,容量为y,x连向可居住的各星球,容量y
每个星球还是连向汇点,容量为容纳上限

POJ3177 Redundant Paths

给出无向图,求出最少增加多少条边后会变为双连通图
双连通分量中,已经是每个点都可互达
于是可以用缩点的思想,把双连通分量进行缩点
缩点后图会变为树
剩下的问题就是最少加多少条边,树会变为双连通图
只要把入度为1的点之间连一条边,那么就会形成环路
一条边可以解决掉2个度数为1的点
于是求出度数为1的点的个数deg1
答案便是deg1/2
但是这种解法有个小BUG
如果只有1个入度为1的点
1/2==0很显然是错的
推广一下,deg1为奇数时,需要(deg1+1)/2
偶数+1,由于是整除,对结果也不会有任何影响
于是正解为(deg1+1)/2
还有个问题需要注意的是,题目中说了,可能会出现重边
于是使用邻接表加边时,需要判重
顺便,POJ3352和本题唯一的不同之处就是POJ3352保证没有重边
于是直接用本题代码提交到POJ3352是完全可以AC的
