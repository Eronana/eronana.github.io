POJ3723 Conscription

##题意
有N个女孩和M个男孩
你需要招募当兵,每招募一个人,需要花费10000块
但是如果你招募了某个女孩,然后这个女孩和某个男孩之间的关系为d
则你再招募这个男孩,只需花费10000-d块,反之也是如此
并且招募每个人,只能使用一次关系
给出所有关系,求最小花费

##解题思路
把每个人看成点,关系看做边
我们需要选择一些边,使这些边的权值之和最大
并且保证由这些边构成的图不存在环(每个人只使用一次关系)
答案就是总人数*10000-边权和
而选择这些边,实际上就是求最大生成树
不过题目中的数据并不会保证是联通的
所有实际上求的是最大生成森林
幸运的是求最小生成树的Kruskal算法可以几乎不加修改的完成
只需把升序排序改成降序排序即可,也就是变成了最大生成树的Kruskal
而对于非连通图,Kruskal的结果直接就是最小(大)生成森林

POJ3308 Paratroopers

##题意
火星人入侵地球啦
有l个火星人在m*n的网格上
我们可以用激光枪消灭一整行或一整列的火星人
在第i行使用激光枪的费用为ri
在第i列是用激光枪的费用为li
总费用为各个费用的乘机
问消灭所有火星人的最少花费

##解题思路
首先先把麻烦的乘积解决,方法是使用对数
log(w1*w2*...*wn)=log(w1)+log(w2)+...+log(wn)
由于对数有这种性质,所以直接对费用取对数
这样就变成比较简单的费用相加了
把行和列看成点,火星人的位置看成边
于是得到了行和列的二分图
由于每个火星人都要消灭,并且每个火星人都可以用行或列的激光枪消灭
反映在图上,就是选择一个可以覆盖所有边的点集,并且使总点权最小
也就是最小点权覆盖
在二分图上,最小点权覆盖=最小割
而最小割=最大流
所以用最大流可以解决

###建图方式
源点连接所有行,容量为从该行发射激光枪的费用的对数
所有列连接到汇点,容量为从该列发射激光枪的费用的对数
然后对于所有火星人的坐标i行j列
将i行连向j列,容量为INF
最后对最大流的之去幂即可

###细节问题
由于是浮点数,所以存图,以及最大流算法中的相应变量都要用double
我的递归版SAP模板误差极大,而且神奇的是解决误差的方法,是把INF设得很小
而且在小得离谱的情况下(1e2)才能得到正确结果
这令我百思不得其解
最后换了个非递归的SAP模板后直接AC...

POJ1325 Machine Schedule

##题意
A机器有n种工作模式,B机器有m种工作模式
两个机器初始摸个都为0
现在有k个任务
第i个任务用A机器的xi模式完成,也可以用B机器的yi模式完成
求完成这k个任务,最少需要切换几次模式

##解题思路
我们把A机器的n个模式看做一个点集A
把B机器的m个工作模式看做一个点集B
然后k个任务看做k条边,每条边连接着两个点集的两个点
于是就构成了一个二分图
我们要求的是最少切换几次工作模式,体现在图上,就是
选择尽可能少得点,使得所有的边,都连在这些点上
也就是最小顶点覆盖
根据Konig定理,二分图的最小顶点覆盖=最大匹配
于是求最大匹配即可
需要注意的是,机器初始为0模式
所以0模式可以处理的任务,就无视掉

CF520C DNA Alignment

##题意
假定字符串s和t的长度都是n,则函数h(s,t)表示位置和字符都相同的个数
然后函数p(s,t)的定义是

shit(s,i)是将s循环移动i位.例如
ρ("AGC", "CGT") = 
h("AGC", "CGT") + h("AGC", "GTC") + h("AGC", "TCG") + 
h("GCA", "CGT") + h("GCA", "GTC") + h("GCA", "TCG") + 
h("CAG", "CGT") + h("CAG", "GTC") + h("CAG", "TCG") = 
1 + 1 + 0 + 0 + 1 + 1 + 1 + 0 + 1 = 6
给出一个字符串s,求有多少个字符串t,满足p(s,t)最大
答案很大,需要对10^9+7取模

##解题思路
首先观察函数p(s,t)
对于每个s,t都移动过不同的顺序,其值之和就是s和t中相同的字符的个数
而s也要移动n次,所以p(s,t)的实质是s和t中相同的字符数*n
有了这个结论就好说了
对于给定的s,使p(s,t)最大,只需让t全部为s中出现最多的字符即可
若s中,有m个字符出现的次数相同且最大
则对于t中的每个字符,为m个字符中的任意一个,对结果都不产生影响
于是t总共有m^n种情况,本题已破
总结一下
计算出s中出现最多的字符的有多少种,记为m
然后使用快速幂计算pow(m,n)%1000000007即可

CF520B Two Buttons

##题意
有一台机器,显示器上显示一个数
有两个按钮,一个将显示的数-1,一个将显示的数*2
如果该数变为负数,则机器会挂掉,所以不能将减为负
问从n到m,最少需要按多少次

##解题思路

###BFS
把每个数都看成点
对于每个数x,连接两条边
x->x-1
x->x*2
然后就是求从n到m得最短路
很明显,边权为1,所以用BFS即可
实现上也不用真正的建图,只需将x-1和x*2入队即可

###贪心
n通过-1和*2到达m,和m通过+1和/2到达n所需的次数是相同的
所以我们从m入手
如果m>n并且m是偶数,则m/=2
如果m<n则m--
最后输出次数即可

POJ3189 Steady Cow Assignment

##题意
N只奶牛要安排到B个牛栏里
每个牛栏都有容量上限
并且每只奶牛对每个牛栏的满意程度都不同
求出最小的范围,令奶牛居住的牛栏的满意程度都在这个范围内

##解题思路
对于最小范围,我们进行枚举
然后将在这个范围内的奶牛和牛栏连边,最后求一下多重匹配
如果多重匹配等于N,则更新答案并缩小范围
如果小于N,则需要扩大范围
实现上,并不需要每次匹配都重建图
加边时,记录该边的满意程度
求匹配时,如果某条边的满意程度不在当前的范围内,就跳过
