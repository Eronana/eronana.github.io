LightOJ1074 Extended Traffic

##题意
由于城市越来越拥挤,于是政府决定收取过路费
每个城市都有个繁忙度busy[i]
如果你的某次旅行经过若干个城市
政府会收取对于所有(busy[v]-busy[u])^3的总和的路费
u->v代表你经过的所有路
现在给出一些起点和终点,最少花费是多少

##解题思路
乍一看是很简单的最短路,不过题目中隐含了一个条件,就是可能存在负边权
若u->v的边权为正,则v->u的边权一定为负
而且可能存在负换的情况,若存在负换,就表示你可以通过旅行,让政府给你无限的钱
鉴于本体的实际背景,这种情况是不可能的
所以说,存在负环,或是总路程为负,就表示你无需交费
于是本体的关键在于找负环,并且把负环上的点的距离标为0
SPFA算法可以找到负环,在遇到负环后,从当前点进行DFS
对于所有可达改点的点,都标为在负环上,以后松弛操作时也需要跳过这些点

POJ3259 Wormholes

##题意
FJ有N个农场,农场之间有M路,K个虫洞
每条路都是双向的,连接着两个农场,通过需要花费一定的时间
每个虫洞都是单向的,也连接着两个农场,虫洞通向过去
FJ向知道,他是否可以从某个农场出发,经过若条路和虫洞后,能早于出发时回到农场

##解题思路
农场看做点,路和虫洞看做边,时间为权值
于是边的权值为正,虫洞的权值为负
能早于出发时回到农场就表示,能回到过去,也就是存在负环
用SPFA找负环即可

POJ2135 Farm Tour

##题意
一个有n个节点的无向图,每条边只能走一次,求从1到n再从n到1的最短路

##解题思路
首先,从1到n,再从n到1,可以看走两次从1到n
于是变成,每条边只能走一次,找出两条从1到n的路径,使其权值之和最小
显然,先求一次最短路,然后删掉走过的边,再求一次最短路的这种方法是错误的
而解决这个问题的方法,便是最小费用流
边的容量为1,这样就能保证每条边只走一次
边的权值为为费用,1为s,n为t,超级源点连向S,容量为2
这样求最小费用最大流即可,最大流很显然为2,即两次

###算法
解决最小费用流的方法很简单
每次沿着费用最小的边进行增广,直至流量最大
由于回退边的费用为负,所以最短路算法使用SPFA
当然也可以用重标号的方法进行修正,便可使用Dijkstra算法

POJ1201 Intervals

略有难度的差分约束题
给出n个如下条件
在[ai,bi]区间内至少选择ci个元素
求出总共最少选择多少个元素可满足约束条件
继续前缀和思想
Ti表示从头[0,i]区间内选择多少个元素
于是条件变为
$$T_bi-T_{ai-1}\geq ci$$
不过题目范围是0 <= ai <= bi <= 50000
0-1就变成-1了,于是这里需要做一下修改
另Ti表示[0,i-1]内选择的数量,则条件变为
$$T_{bi+1}-T_ai\geq ci$$
由于是大于等于,所以就变为求最长路径了
不够目前条件还不够,还需要增加条件
$$0\leq T_{i+1}-T_i\leq 1$$
这个不等式拆成两个,总共3个条件
然后建图,SPFA即可
其实次题用Dijkstra也可以
因为如果把区间内所有元素都选择的话,一定满足约束条件
也就是说,至少会有这1种解,即不存在负权(对应最长路是正权)回路

还差个等号,由于是整数,直接k-1即可,于是

关于Dijkstra的堆优化算法

Dijkstra也写了很多次了
今天看了下在HicoCoder上的别人的Dijkstar
发现比我写的快了好多,我的76ms,别人的0ms,1ms
这也差的太多了吧
于是开始对比,究竟是哪里的问题
于是发现,我是这么写的

而别人的,是这样的

作为Dijkstra的核心代码,写法却有些差异
首先说前者,也就是我惯用的写法,慢的原因是没有记录已经完成的点
当一个点被松弛多次后最后松弛的会第一个出队,然后松弛临近的点
完成后,间隔若干次,同一个点再次出队,遍历一遍邻接的点(仅仅是遍历,不会再松弛)
然后..还有第三个,如果被松弛了3次的话
于是很显然,浪费了效率
后者,使用了vis数组记录访问过的点,就没有这种缺陷,速度自然就上去了
遍历到v,直接就吧v丢进q里也是一种效率的浪费,不过似乎影响不大
不过,其实这点差距引起的效率,也不足以引起我的注意
实际上引起效率巨大差距的原因是....我写错了比较函数
于是变成了大顶堆orz没超时就不错了
不过变成了大顶堆的情况,为什么也能得到正确结果呢?
你看,SPFA就没有用优先队列,不也一样能求出来么
比较一下不考虑负环情况的SPFA和优先队列的Dijkstra

SPFA Dijkstra
队列 优先队列
不记录和判断已访问过的点 记录和判断已访问过的点

还有其他区别吗?似乎没有了.
于是即使是写错变成大顶堆的优先队列,由于判断条件是队列不为空就继续松弛下去
于是总能完成算法并保证正确性..
所以,以后写Dijkstra时,为了保证正确性和效率,都要加上判断...
不然犯了写错符号这种低级错误时都不能立刻发现..
