CF534D Handshakes

##题意
一个选手来到训练场会与所有没有训练的选手握手
在任意时刻,都有可能出现三个没有训练的选手组成一队开始训练
给出每个选手与其握手的人数,尝试找出所有选手入场的顺序

##解题思路
首先从与0个人握手的选手开始,很明显他是第一个到来的
然后寻找是否存在与1个人握手的,接着是2个..
接下来,有两种情况
如果这三个人开始组队训练,那么第四个人到场时将与0个人握手
否则的话,第四个人会与三个人握手
到第7个人的时候,也是这种情况,可能有1组训练的,也可能有两组训练的,也可能都没有训练
这样对于第7个人来说,就有0 3 6三种情况
可以发现,每3个人一循环..
也就是说..我们只要找是否存在与p%3个人握手的人即可
p表示当前的人数
若存在与p个人握手的人,那么p++
若不存在,则表示有人组队训练了,找是否存在p-3,p-6...并且要尽可能的大
如果还找不到,就说明无解,输出Improssible

##实现细节
实现上,可以使用一个长度为3的set,保存握手次数%3的每个人的握手次数和编号
然后查找时,用lower_bound查找>=p的元素x
如果x的握手次数等于p,那么直接输出编号
否则的话,x一定是第一个>p的元素,如果存在<p的元素则,x前面一定存在其他元素
也就是x!=begin(),如成立,输出编号
如果x==begin(),则表示x前面没有其他元素,也就是不存在握手次数<p的元素,也就是无解
输出编号后,需要维护p,p=握手次数+1,然后删除x

CF534C Polycarpus' Dice

#题意
有n个骰子,地i个骰子可投掷出1-di
现在给出n个骰子的值的和A
问每个骰子不可能的值有多少个
2 3
2 3
两个骰子,总和是3
第一个1的话,第二个就2
第一个2的话,第二个就1
所以第一个骰子没有不可能的值
第二个同上,可为1,可为2,但是不可为3,于是不可能的值的个数为1
最终输出0 1

##解题思路
对所有di求和的S
很显然,可投掷出的总值的区间为[n,S]
对于第i个骰子,若掷出1,则最大值为S-di+1
若掷出di,则最小值为n+di-1
也就是说,若a>S-di+1,则必须依靠第i个骰子来调节才可能使总和为a
换句话说,就是第i个骰子就会出现不可能的值
于是很容易得出,不可能取值的个数为a-(S-di+1)
对于a<n+di-1的情况也如此
总结一下
若a>S-di+1,则有a-(S-di+1)个值不可取
若aa<n+di-1,则有n+di-1-a个值不可取
ps:2*10^5个10^6相加会爆int,所以要用long long

POJ3169 Layout

##题意
N个奶牛按照从1到N的顺序排队,并且奶牛们会有这两种要求
A奶牛和B奶牛的距离不能超过D
A奶牛和B奶牛的距离不能小于D
求第一只奶牛和第N只奶牛的最大距离
若无法达成奶牛的要求,输出-1
如果1和N的距离可以无限远,输出-2

##解题思路
很明显的差分约束系统
求最大距离,就要求最短路
根据两个条件,添加相符的边即可
不过本题有一个蕴含条件就是相邻的两只奶牛的距离要>0,因为是按照从1到N的顺序排队的
而差分约束系统除了有确定的解外,还有其他两种情况
无解和任意多的解,也就是对应这本题的另外两种输出
当无解时,会出现负环,当有无限个解时,最短路为INF

HDU4725 The Shortest Path in Nya Graph

##题意
有N个点,每个点都属于一个层
可以花费C,到达相邻层的任意位置
还有额外M条边,可以通过额外的边花费w从一个点到达另一个点
问从1点到达n点的最小花费

##解题思路
一个建图略麻烦的最短路问题
m条边就不用说了,直接添加为双向边就好
而对于这个层,若a,b,c在L1层,d,e,f在L2层
L1层和L2层是相邻的,则a,b,c任意一点到达d,e,f的任意一点的费用都是C
若直接这样加边,边数为两层点数相乘/2,非常大
所以就需要用拆点的办法,拆点的方法在网络流里比较常用
对于每层i,都设立一个点Si,所有点到这点的费用都是0
然后相邻的两层i,j,添加边Si到Sj,费用为C
通过这种把路径统一的方法,就可以解决边数过多的问题
不够单纯这么做是错误的,也是初学拆点常犯的错误
若某层中有两点a,b.那么有边a-S,b-S,费用都为0
于是问题来了,通过a-S-b这条路劲,a-b的费用就变成了0,这显然是不对的
于是很显然,我们需要添加的是有向边,只能a->S,b->S,而不能往回走
这就需要把两个点,S和T
a->S,b->S,T->a,T->B
对于每层的S,与相邻的层的T连有向边
通过这种限制方向的方法,即可解决上述问题
建好图后,直接跑最短路即可解决本题

LightOJ1074 Extended Traffic

##题意
由于城市越来越拥挤,于是政府决定收取过路费
每个城市都有个繁忙度busy[i]
如果你的某次旅行经过若干个城市
政府会收取对于所有(busy[v]-busy[u])^3的总和的路费
u->v代表你经过的所有路
现在给出一些起点和终点,最少花费是多少

##解题思路
乍一看是很简单的最短路,不过题目中隐含了一个条件,就是可能存在负边权
若u->v的边权为正,则v->u的边权一定为负
而且可能存在负换的情况,若存在负换,就表示你可以通过旅行,让政府给你无限的钱
鉴于本体的实际背景,这种情况是不可能的
所以说,存在负环,或是总路程为负,就表示你无需交费
于是本体的关键在于找负环,并且把负环上的点的距离标为0
SPFA算法可以找到负环,在遇到负环后,从当前点进行DFS
对于所有可达改点的点,都标为在负环上,以后松弛操作时也需要跳过这些点

POJ1062 昂贵的聘礼

##题意
中文题意..直接复制过来好了

年轻的探险家来到了一个印第安部落里。在那里他和酋长的女儿相爱了，于是便向酋长去求亲。酋长要他用10000个金币作为聘礼才答应把女儿嫁给他。探险家拿不出这么多金币，便请求酋长降低要求。酋长说："嗯，如果你能够替我弄到大祭司的皮袄，我可以只要8000金币。如果你能够弄来他的水晶球，那么只要5000金币就行了。"探险家就跑到大祭司那里，向他要求皮袄或水晶球，大祭司要他用金币来换，或者替他弄来其他的东西，他可以降低价格。探险家于是又跑到其他地方，其他人也提出了类似的要求，或者直接用金币换，或者找到其他东西就可以降低价格。不过探险家没必要用多样东西去换一样东西，因为不会得到更低的价格。探险家现在很需要你的帮忙，让他用最少的金币娶到自己的心上人。另外他要告诉你的是，在这个部落里，等级观念十分森严。地位差距超过一定限制的两个人之间不会进行任何形式的直接接触，包括交易。他是一个外来人，所以可以不受这些限制。但是如果他和某个地位较低的人进行了交易，地位较高的的人不会再和他交易，他们认为这样等于是间接接触，反过来也一样。因此你需要在考虑所有的情况以后给他提供一个最好的方案。
为了方便起见，我们把所有的物品从1开始进行编号，酋长的允诺也看作一个物品，并且编号总是1。每个物品都有对应的价格P，主人的地位等级L，以及一系列的替代品Ti和该替代品所对应的"优惠"Vi。如果两人地位等级差距超过了M，就不能"间接交易"。你必须根据这些数据来计算出探险家最少需要多少金币才能娶到酋长的女儿。

##解题思路
把每个物品看做点,B物品+V元换得A物品看做A->B的边
于是问题就变成了求1到各点的最短路中,到某点最短路+某点物品价格的最小值
还有一个问题就是等级限制,我们可以枚举可以用的区间,然后只对区间内的点做最短路即可
