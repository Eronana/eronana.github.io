HDU4738 Caocao's Bridges

##题意
曹操在长江上建了N个岛,用M个桥相连
周瑜摧毁一些桥,使得其中的一些岛与其他的岛分离
但是周瑜只有一个炸弹,只能炸毁一座桥
并且每个桥上都有卫兵把守,周瑜必须派出不小于卫兵的人数才能用炸弹炸毁某个桥
问周瑜最少需要多少人才能达到目的
如果无法达到目的,输出-1

##解题思路
很明显是找出权值最小的桥
这里的桥不是题中的桥,而是连接双连通分量的桥
不过本体有两个坑点
1.如果求得答案为0,则要输出1,因为至少需要1个人携带炸药炸掉
2.可能会包含重边,所以不能用判断父节点的方法来避免走重,要判断边

HDU4685 Prince and Princess

##题意
基本和POJ1904 King's Quest一样
区别是本题的王子和公主的数量并不相同,并且没有给匹配列表

##解题思路
于是首先应该求王子和公主的最大匹配
不过由于数目不同,所以并不能达成完美匹配(即使是相同,也不一定是完美匹配)
根据POJ1904的连通分量内可任意互换的思想
如果某个人没有匹配,则无法求出其可能的配偶
所以我们在求出最大匹配后,为没有匹配的人添加虚拟配偶
如果一个王子没有匹配,则添加一个虚拟公主
虚拟公主向该王子连边,所有王子向虚拟公主连边
如果一个公主没有匹配,则添加一个虚拟王子
公主向虚拟王子连边,王子向所有公主连边
剩下的就通POJ1904一样处理即可,只是最后要判断一下,不要输出虚拟的人即可

HDU4635 Strongly connected

##题意
给一个非强连通图,问最多能添加多少边,能让这个图仍然不是强连通图

##解题思路
逆向思维,对于完全图,最少去掉多少条边可使其不联通
首先把原图的强联通分量缩点,然后找出分量内点数最少且比较孤立的分量
把除了这个分量外的点全部相互连边
所谓比较孤立,是指分量与外部没有入边或出边
把分量外的点相互连边后,外部的这些点就组成了一个大的强连通分量
如果与该分量既有出边又有入边,那么这两个分量就令整个图连通
如果没有出边或入边或两者都没有,那么这两个分量之间就不会强连通,也就不会让整个图连同
最后答案为
完全图的边数-已有的m条边-最小孤立分量与其他点之间的边数

POJ1904 King's Quest

##题意
国王有N个儿子,王国里有N个妹子
国王知道他每个儿子喜欢的每个妹子
国王想让他的每个儿子都能与喜欢的妹子结婚
于是巫师制作了一个分配列表,上面列出了国王的每个儿子,以及分配的妹子
并且清单保证,国王的每个儿子分配到的妹子都是其喜欢的
但是国王并不满足于此,国王想知道能保证每个儿子都能与其喜欢的妹子结婚的前提下
列出每个儿子所有可选的妹子

##解题思路
儿子向喜欢的妹子连有向边,列表中的妹子向儿子连有向边
很显然,边表示的是可选的异性
然后求强连通分量
设第i个儿子被分配的妹子的编号是k
如果在分找到一个儿子i喜欢的妹子j,重新分配给儿子i
那么之前分配到妹子j的人就需要重新找其他的妹子了....
于是重新为i分配,会引起接下来一系列的重新分配..
分配方向是沿着儿子i->妹纸j->妹子j之前分配给的某儿子->某妹子->某儿子
由于是在强连通分两中,根据强联通分量的定义,分量内每个点都是可互达的
于是最后总能回到k,也就是说,一定可以完成重新分配
于是对于分量中的的儿子和妹子,按照这种方法,都可以匹配
于是每个儿子的可选妹子,就是其所喜欢的,并与其在同一分量中的妹子

关于并查集的初始化

主流的并查集是这样初始化的

然后Find函数这样定义
int Find(int a)
{
return Father[a]==a?a:Father[a]=Find(Father[a]);
}
一开始我也这么用,直到某天突然发觉,可以这样初始化

然后Find函数这样定义
int Find(int a)
{
return Father[a]==-1?a:Father[a]=Find(Father[a]);
}
也就是说,改变了根的标识
前者i为根时,Father[i]==i
二后者i为根时,Father[i]==-1
于是通过这样的小修改,便可以让初始化简单一点
不过也带来了一些不方便的地方
如果用前者,在全部合并操作完成后,对所有元素查找一遍

然后就可以用Father[i]来表示i的根了
理由很简单,如果i是根,那么Father[i]一定是i
如果i不是树根,那么经过之前的Find(i)路径压缩后,Father[i]便是i的根
不过这种技巧对于后者来说则行不通
因为后者用Father[i]==-1来表示i是根

HDU4612 Warm up

##题意
给一个无向图,怎样添加一条边,可使桥的数量最小
输出最小的桥的数量

##解题思路
首先要求出桥的数量,已经双连通分量
把分量缩点,于是原图变为树,桥就是边
现在需要连一条边,然后会形成环,我们需要的是让环的边数尽可能多
很显然,找到距离最远的两点,连边即可
而这最远的两点就是树的直径的两端,两次BFS/DFS即可求出
需要注意的是,本题可能有重边
于是求分量时不能判断父节点,而是要判断边是否走过
而且本题数据了略大,需要扩栈,否则Tarjan时会爆栈
实现上,找到桥后,存起来
Tarjan算法完成后,用找到的桥建新图(树),并且直接用分量缩点
用两次BFS/DFS求出树的直径
最终答案为桥的数量减去树的直径

##一些领悟
关于求边双连通分量,我之前写过一篇关于实现上的问题的博文
关于求边的双连通分量
今天在看别人的代码时发现,在判断桥的同时取出分量也可以争取
而且我之前的分析,这样做却有问题
最后会省下一个分量留在栈里,详见上面提到的博文
最后仔细分析了代码弄懂了
因为只有一个分量取不到
于是分量分量数组初始为0
取分量时,从1开始计数
最后省下的那个没取到的分量,默认为0,于是算法正确
对于这两种方法,找桥时直接取,需要清空栈,初始化分量数组
而遍历完边再取分量,则不需要清空栈,因为肯定会取完,也不需要初始化,因为肯定会把每个节点都填上分量编号

POJ3694 Network

##题意
给一个无向图,动态的加边,输出每次加边后桥的数量

##解题思路
先用Tarjan求出所有桥,并把双连通分量缩点,于是原图就变成了树
如果加边不能减少桥,那么这条边的两个点一定位于同一个分量中
如果可以减少桥,那么在树上肯定会形成环
换上的边都是减少的桥,只要找出这些边即可
很容易得出,添加u->v边后形成了环,那么这个环的最上方一定是lac(u,v)
用样例2来举例

在3,4直接连边,3,4,lca(3,4)=1会形成环
在考虑缩点的话,1 2 3直接缩成一点,于是把1,2,3都当成1
3,4连边,相当于1,4连边,于是1,4,lca(1,4)=1形成环
只要动态的找出需要连边的u,v两点的LCA,并相应的缩点即可
实现上,有两种方法
一种是不缩点,直接在原图上进行LCA
另一种是缩点,在求桥时,直接用并查集把在同一分量中的点并在一起,并切要让最上方的点为根
求LCA时,用并查集就可以快速找到当前分量的最上方的点(并查集的根)
然后向上回溯即可

POJ3592 Instantaneous Transference

##题意
1 //一组数据
2 2 //下面是2*2的地图
11 //数字代表该点的矿物数量
1* //*代表传送点
0 0 //传送点的目标坐标
地图中可能有多个传送点
地图上还可能有#,表示不能进入的区域
从左上角开始走,只能向下走或向右走
到达的点上的的矿物会被取走
到达传送点,可以选择传送到目标坐标,或者继续向下或向右走
问最多能挖到多少矿物

##解题思路
如果没有传送点,本题就是一道十分简单的搜索(DP)
因为存在传送点,就构成了环路,所以不能直接搜索或DP
很容易想到,既然形成了环路,那么环路上的点一定都可以取到,并且到达环路上的任意一点
而且多个环路套在一起也是如此,于是很容易就能想到缩点
先求出强连通分量,然后把每个分量缩成一点,矿物数量为分量所有点之和
缩点后,环路自然消失,可以安心的搜索/DP了
