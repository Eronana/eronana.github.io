CF540D Bad Luck Island

##题意
The Bad Luck Island有三个部落:石头,剪刀,布
每两个人碰到的概率是相同的
当石头部落的人碰到剪刀部落的人,剪刀部落的就会被石头部落的杀死
剪刀和布,布和石头也如此
于是很显然,经过足够多的碰面后,最终会剩下一个部落
给出三个部落的人数,求出这三个部落活到最后的概率

##解题思路
先采用分治的思想,缩小问题规模
于是{r,s,p}=
布碰到石头的概率*{r-1,s,p}+
石头碰到剪刀的概率*{r,s-1,p}+
剪刀碰到布的概率*{r,s,p-1}
边界情况是,当某部落人数为0时,某一部落获胜的概率就是1
两个部落相遇的概率就是这两个部落的总人数/三个部落的总人数
实现上,写成dfs的形式的记忆化搜搜即可(其实是我不擅长写for形式的DP)

CF538C Tourist's Notes

##题意
爬山日志记录了每天的高度:第i天的高度为$h_i$
并且由于选择了十分平坦的道路,两天的高度只差不会大于1
爬山结束后回到家,发现日志丢失了很多,只剩下一部分
现在你需要根据这些日志找出可能到达的最高的高度
如果日志记录有误,输出IMPOSSIBLE

##解题思路
这题比较偏数学,解法是列两个不等式
若i和j是残缺日志中相邻的两天
设第k天是第i天到第j天中最高的那天,则有
$
\begin{cases}
h_k \leq h_i+(k-i) \
h_k \leq h_j+(j-k)
\end{cases}
$
解得
$h_k \leq \frac{h_i+h_j+j-i}{2}$
有了公式就简单了,开头和结尾单独计算,然后对所有i,j,算出$h_k$取最大值即可
最后是判断IMPOSSIBLE,如果$\left|h_j-h_i|\right>j-i$就输出IMPOSSIBLE

CF538B Quasi Binary

##题意
给你一个数字,你需要把它分解成最少的01数字相加的形式
01数字是指只包含0和1的十进制数字

##解题思路
基于贪心思想,比如35427
对于万位3来说,我们肯定需要三个1xxxx
然后千位5,我们让之前的三个1xxxx变成11xxx
这样还缺少两个千位,所以需要两个1xxx
再看百位4,由于之前已经有五个可分配的x了,所以只需呀让其中四个为1,其中一个为0
比如这样
111xx
111xx
111xx
11xx
10xx
然后是十位,是2,和百位一样,依法炮制即可
最后的各位是7,之前的5个的各位都设为1,也只能5,所以还需要增加两个1
于是最终答案就是
11111
11111
11101
1101
1001
1
1

##实现细节
对于如上过程,可以发现并不需要读入成整数,而是直接用字符串处理即可

HDU5211 Mutiple

##题意
wld有一个序列$a[1..n]$, 对于每个$1 \leq i < n$, 他希望你求出一个最小的$j$（以后用记号$F(i)$表示），满足$i < j \leq n$, 使$aj$为$ai$的倍数（即$aj$ mod $ai$＝0），若不存在这样的$j$，那么此时令$F(i)$ = 0
保证$1 \leq n \leq 10000, 1 \leq ai \leq 10000$ 对于任意 $1 \leq i \leq n$, 且对于任意$1 \leq i, j \leq n(i!=j)$，满足$ai$ != $aj$

##解题思路
由于保证了每个数都不同,所以很好搞
读入每个数a[i]的同时,记录这个数的位置x[a[i]]
读完后开始扫一遍
对于每个数j,从a[i]*2开始,直到n,每次增加a[i],然后找出大于i的最小的x[j]
若存在,x[j]就是f(i),累加到ans即可
